Moving Pattern Analysis of Data Collected from Personal Activity Monitoring Device

This is an educational project, which is the part of Data Science Specialization provided by Johns Hopkins University on Coursera.

Introduction

It is now possible to collect a large amount of data about personal movement using activity monitoring devices such as a Fitbit, Nike Fuelband, or Jawbone Up. These type of devices are part of the quantified self movement - a group of enthusiasts who take measurements about themselves regularly to improve their health, to find patterns in their behavior, or because they are tech geeks. But these data remain under-utilized both because the raw data are hard to obtain and there is a lack of statistical methods and software for processing and interpreting the data.

Data

The data for this assignment can be downloaded from the Coursera web site:

  • Dataset: Activity monitoring data [52K]

It consists of data from personal activity monitoring device. This device collects data at 5 minute intervals through out the day. The data consists of two months of data from an anonymous individual collected during the months of October and November, 2012 and include the number of steps taken in 5 minute intervals each day.

The variables included in this dataset are:

  • steps: Number of steps taking in a 5-minute interval (missing values are coded as NA)

  • date: The date on which the measurement was taken in YYYY-MM-DD format

  • interval: Identifier for the 5-minute interval in which measurement was taken

The dataset is stored in a comma-separated-value (CSV) file and there are a total of 17,568 observations in this dataset.

What are the statistical parameters of the number of steps taken per day?

In order to obtain mean and median number of steps per day was summed up data for all 5 minute intervals for each date. Then mean and median were calculated and was built a histogram of the total number of steps taken per day in October and November 2012.

day_summary <- tapply(data$steps, data$date, sum, na.rm=TRUE)

What is the average daily activity pattern?

The average number of steps taken for each 5 minute time interval throughout a day was calculated.

time_summary <- tapply(data$steps, data$interval, mean, na.rm=TRUE)
plot(as.numeric(names(time_summary)), time_summary, type="l", cex.lab=1.3,  
     ylab="Average number of steps", xlab="Day timeline, minutes", 
     col="steelblue",  
     main=paste("Average number of steps throughout the day",  
                "from \n anonymous individual. Collected in",  
                " \nOctober-November, 2012"))

Then we found the 5 minute time interval where the number of steps is maximal on average throughout the day.

max_steps <- as.numeric(names(which.max(time_summary)))
max_steps_hours <- max_steps %/% 60
max_steps_minutes <- max_steps %% 60

The maximal average number of steps was for 835 minutes time interval, which corresponds time between 13:50 and 13:55.

Imputing missing values

The total number of missing values in the dataset (i.e.the total number of rows with NAs) was calculated.

sum(!complete.cases(data))
## [1] 2304

There are 2304 rows with NA in dataset.

In order to impute NA values we created a new dataset that was equal to the original dataset but with the missing data filled in with mean for 5-minute interval that corresponds to given missing value.

data_filled <- data
for (x in 1:length(data_filled$steps)) {
    if (is.na(data_filled$steps[x])) {
        data_filled$steps[x] = time_summary[as.character(data_filled$interval[x])][[1]]
    }
}

Are there differences in activity patterns between weekdays and weekends?

At the next stage we exemined if the pattern of number of steps is differ for weekdays and weekends. At first we created a new variable weekday, which is a factor with two levels weekday and weekend.

weekday <- factor(as.POSIXlt(data_filled$date)$wday == 0 | as.POSIXlt(data_filled$date)$wday == 6,  
                  labels=c("weekday", "weekend"))
data_filled$weekday <- weekday
